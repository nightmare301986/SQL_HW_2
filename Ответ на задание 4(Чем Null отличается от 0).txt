В SQL null или NULL - это специальный маркер, используемый для указания того, что значение данных не существует в базе данных. 
Введенный создателем реляционной модели базы данных Э. Ф. Коддом SQL null служит для выполнения требования о том, 
что все настоящие системы управления реляционными базами данных (СУБД) поддерживают представление "отсутствующей информации 
и неприменимой информации". Кодд также ввел использование строчного греческого символа омега (ω) для обозначения null 
в теории баз данных. В SQL NULL это зарезервированное слово, используемое для идентификации этого маркера.

Значение null не следует путать со значением 0. Значение null указывает на отсутствие значения, что не то же самое, 
что значение, равное нулю. Например, рассмотрим вопрос "Сколько книг принадлежит Адаму?" 
Ответом может быть "ноль" (мы знаем, что он не владеет ни одним) или "ноль" (мы не знаем, сколькими он владеет). 
В таблице базы данных столбец, сообщающий об этом ответе, начинался бы со значения no (помеченного Null), 
и он не обновлялся бы значением "ноль" до тех пор, пока мы не убедились бы, что Адам не владеет книгами.

SQL null - это маркер, а не значение. Это использование сильно отличается от большинства языков программирования, 
где нулевое значение ссылки означает, что она не указывает ни на какой объект.
А число "0" это законный участник всех арифметических операций.
